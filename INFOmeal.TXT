Receptų paieškos web aplikacija
Tikslas-Sukurti React aplikaciją, kuri leidžia ieškoti patiekalų (meals) per TheMealDB API, filtruoti rezultatus ir peržiūrėti detales.
UI turi atitikti pateiktą Figma dizainą, būti modernus, responsyvus ir su tvarkinga komponentų/ modulių struktūra.
Figma dizainas:
https://www.figma.com/design/EClchxUSyjjjZWlT3TGo6I/Recipedia---Landing-Page--Community-?node-id=0-1&t=2oPSu4lmQqp56o9T-1

1. Funkcionalumas (React)
1) Pagrindinis puslapis (Home)
Viršuje search bar (paieška pagal: pavadinimą, ingredientą arba kategoriją).
Rodomas kategorijų sąrašas (navigacijai / greitam pasirinkimui).
2) Paieškos rezultatų puslapis (Search Results)
Po paieškos rodomas sąrašas kortelėmis:, patiekalo pavadinimas, nuotrauka, trumpas aprašymas (category/area info)
Jei nieko nerasta – aiški “No results” žinutė.
Loading būsena + error būsena.
3) Patiekalo detalių puslapis (Meal Details)
Atsidaro paspaudus ant patiekalo kortelės.
Rodyti: didelė nuotrauka, ingredientų sąrašas (su kiekiais), instrukcijos, category, area, tags (jei yra)
4) Random Meal sekcija / puslapis
Mygtukas “Random meal” Paspaudus – užkrauna atsitiktinį patiekalą (galima rodyti kaip kortelę).
 
2. TheMealDB API reikalavimai: Endpoint’ai:
Paieška pagal pavadinimą
https://www.themealdb.com/api/json/v1/1/search.php?s=Arrabiata
Pagal pirmą raidę
https://www.themealdb.com/api/json/v1/1/search.php?f=a
Detalės pagal ID
https://www.themealdb.com/api/json/v1/1/lookup.php?i=52772
Random
https://www.themealdb.com/api/json/v1/1/random.php
Filtrai:
ingredient: https://www.themealdb.com/api/json/v1/1/filter.php?i=chicken_breast
category: https://www.themealdb.com/api/json/v1/1/filter.php?c=Seafood
area: https://www.themealdb.com/api/json/v1/1/filter.php?a=Canadian
Vaizdai-Naudoti API grąžinamus image URL.

3. React architektūra: Routing (rekomenduojama su react-router-dom)
Minimalūs route’ai:
/ – Home
/search – Search results (su query parametrais, pvz. ?q=chicken&type=name)
/meal/:id – Meal details
/random – Random meal (arba Random sekcija Home puslapyje)
 
4. HTML5 semantika ir prieinamumas:
Naudok semantinius elementus: <header> <nav> <main> <section> <article> <footer>
Formoms: label + input su htmlFor/id
Mygtukams: aiškūs aria-label, jei reikia
Kortelės kaip <article> (rekomenduojama)
Klaviatūros navigacija (tab) turi veikti

5. Stilius ir responsyvus dizainas:
Dizainas pagal Figma (spalvos, tarpai, tipografija, grid).
Responsive:
mobile first arba su media query
kortelės išsidėsto 1 stulpeliu mobile, 2–4 desktop (pagal plotį)
vaizdai prisitaiko (max-width: 100%)
 
6. Error handling:
Jei API grąžina meals: null → rodyti “No meals found”.
Network error → rodyti “Something went wrong”.
Loading būsena (spinner / skeleton).

Bootstrap naudojamas kaip UI pagalbinė biblioteka (grid, responsive layout, spacing), o visa aplikacijos logika, API integracija, routing ir state management įgyvendinti su React.
